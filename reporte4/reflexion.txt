Realizar esta actividad con XML, XSL y CSS me permitió comprender de manera práctica cómo se puede dar formato y estilo a la información estructurada. El archivo XML actúa como la base de datos en forma de texto, donde están definidos los libros y sus atributos. Sin embargo, por sí solo es solo información sin presentación visual. Es aquí donde entran las otras dos herramientas que fortalecen el aprendizaje: XSL y CSS.
Por un lado, aprendí que XSL no solo se utiliza para transformar el contenido de XML, sino que también permite realizar una especie de query o consulta sobre la información. Esto hace posible seleccionar, filtrar y mostrar únicamente los datos que se necesitan, como en este caso al recorrer cada libro del catálogo. De esta forma, XSL se convierte en un puente que conecta los datos puros con la lógica de presentación.
Por otro lado, comprendí que CSS es la herramienta que le da el diseño y la identidad visual a la información. Gracias a CSS, el catálogo deja de ser una lista plana y pasa a ser un sistema visualmente atractivo, ya sea en forma de tarjetas, lista o con colores diferenciados. Esto refuerza la importancia de separar la estructura de los datos (XML), la lógica de transformación (XSL) y la capa de presentación (CSS).
Como estudiante de Ingeniería en Tecnologías Computacionales (ITC), esta actividad es valiosa porque muestra cómo integrar distintas tecnologías para lograr un resultado completo. No se trata solo de programar, sino de entender cómo se organizan, consultan y presentan los datos. Este tipo de prácticas ayudan a reforzar las bases que después se aplican en proyectos más grandes de integración, bases de datos y desarrollo web.
En conclusión, esta actividad me permitió ver la importancia de usar XML como fuente de datos, XSL para consultas y transformaciones, y CSS para el diseño, entendiendo cómo cada herramienta cumple un rol clave en el manejo y presentación de información.
